# reapp-protocol-cli

Command-line tool for the **REAPP MandateRegistry** — agent payments on Stellar,
enforced on-chain. A user signs an AP2 mandate with a budget; an agent pays for
resources against it; the Soroban contract enforces scope, budget, expiry, and
replay at consume time. The CLI is a thin, untrusted client — the contract is the
source of truth.

Testnet only. Keys generated by the CLI are throwaway testnet burners, written
outside any repo, and must never be reused on mainnet.

## Install

```sh
# one-off
npx reapp-protocol-cli <command>

# or install the `reapp` command
npm i -g reapp-protocol-cli
reapp <command>
```

## Quickstart

```sh
reapp init             # scaffold reapp.config.json in the current directory
reapp setup            # generate + fund testnet keys (user / agent / merchant)
reapp mandate create   # register an AP2 mandate on-chain + grant the allowance
reapp pay              # agent-signed payment; budget enforced on-chain
```

`init` writes a committable `reapp.config.json` (network, the live contract id,
explorer, demo price/budget). `setup` writes secrets to `~/.reapp/credentials.json`
(dir `0700`, file `0600`) — never to the repo. `mandate create` stores the mandate
in `~/.reapp/mandate.json` so `pay` rebuilds the identical mandate id.

## The "aha" in one command

```sh
reapp demo research-agent
```

Self-contained and runs cold: it funds three ephemeral testnet accounts, registers
a mandate with a 3 XLM budget, and an agent buys research sources one at a time —
each a real on-chain `execute_payment`. The budget covers three; the contract
rejects the fourth. No LLM key required. See it running live in your browser at
[reapp.live/t2/demo](https://reapp.live/t2/demo).

## Commands

| Command | What it does |
| --- | --- |
| `reapp init [-f]` | Write `reapp.config.json` in the current directory. |
| `reapp setup [-f]` | Generate three testnet burners and fund them via friendbot. |
| `reapp mandate create [-b <xlm>] [-e <seconds>] [-f]` | Register an AP2 mandate on-chain and grant the SEP-41 allowance to the contract. |
| `reapp pay [amount]` | Make an agent-signed payment against the active mandate. |
| `reapp demo research-agent` | Run the full budget-capped flow end-to-end on testnet. |

## How it works

The user grants the SEP-41 allowance to the **contract**, never to the agent or
the SDK. Money moves only through `MandateRegistry.execute_payment`, which
validates-and-consumes the mandate atomically before it transfers. A compromised
agent or SDK cannot exceed the mandate — `reapp pay` past the budget is rejected
on-chain, not by the CLI.

Ships as a self-contained bundle of `@reapp-sdk/core` and `@reapp-sdk/stellar`
(the on-chain client layers); only `@stellar/stellar-sdk` is an external runtime
dependency.
